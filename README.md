# 물류 입출력 시뮬레이션 (Logistics I/O Simulator)

웹/모바일에서 동작하는 물류 시뮬레이션 프로젝트입니다.  
장비 배치, 물품 이동 속도, 처리 속도, 전력 등을 고려한 시뮬레이션을 지원합니다.

---

## 목표

- **웹·모바일 공통**: 반응형 웹으로 데스크톱·모바일 모두 사용 가능
- **1차**: 고정된 핵심 기능만 구현 후, 단계적으로 기능 추가
- **두 가지 모드**
  - **배치·테스트**: 단순 배치 및 동작 테스트
  - **시뮬레이션**: 시간에 따른 흐름 분석 및 결과 확인

---

## 핵심 요소 (시뮬레이션 파라미터)

| 구분 | 설명 | 예시 |
|------|------|------|
| 장비 배치 | 컨베이어, 로봇, 창고 등 위치/방향 | 그리드 또는 좌표 기반 |
| 물품 이동 속도 | 컨베이어·AGV 등에서의 이동 속도 | m/s, 칸/초 등 |
| 장비 처리 속도 | 입고/출고/가공 처리량 | 개/분, 회/시간 |
| 필요 전력 | 장비별 전력 소비 | kW, 누적 전력 |
| (추가) | 대기열, 병목, 우선순위 등 | 이후 단계에서 정의 |

## 개발 단계 (로드맵)

### Phase 1 – 최소 구현
- [ ] 반응형 레이아웃 (웹 + 모바일)
- [ ] **배치 모드**: 그리드 위에 장비 타입별 배치/삭제/회전
- [ ] **테스트 모드**: 배치된 경로 따라 물품 1개 흐름 확인
- [ ] 시뮬레이션 모드: 고정 시나리오 1~2종 (예: 입고→보관→출고)
- [ ] 기본 지표: 처리량, 평균 대기 시간 (간단 수치만)

### Phase 2 – 시뮬레이션 강화
- [ ] 시간에 따른 그래프/차트 (처리량, 대기열, 전력)
- [ ] 시나리오 저장/불러오기 (JSON 등)
- [ ] 장비별 파라미터 UI (속도, 전력, 용량)

### Phase 3 – 확장
- [ ] 다양한 장비 타입·동작 추가
- [ ] 병목 분석, 최적 배치 제안(휴리스틱)
- [ ] 다국어·테마 등 운영 편의 기능

## 모드 설명

### 1. 배치·테스트 모드
- **목적**: 레이아웃 설계와 단순 동작 검증
- **기능**
  - 맵(그리드) 위에 장비 배치/이동/삭제
  - 연결 관계(입구↔출구) 시각화
  - “한 번에 한 물품” 테스트 실행으로 경로만 확인
- **결과**: 배치가 논리적으로 맞는지, 경로가 끊기지 않는지 확인

### 2. 시뮬레이션 모드
- **목적**: 시간 흐름에 따른 성능·흐름 파악
- **기능**
  - 시뮬레이션 시간(예: 8시간) 동안 입고/출고 이벤트 반영
  - 장비별 처리 속도·전력·대기열 반영
  - 실시간 또는 스텝별 재생/일시정지
- **결과**: 처리량, 평균 대기 시간, 전력 사용량, 병목 구간 등

## 기술 스택 (제안)

- **프레임워크**: React + TypeScript
- **빌드**: Vite (빠른 개발·HMR)
- **스타일**: CSS Modules 또는 Tailwind (반응형·테마 확장 용이)
- **상태**: React 상태로 시작 → 필요 시 Zustand/Context
- **차트(Phase 2)**: Lightweight Charts 또는 Recharts

## GitHub에 올리고 웹에서 접속하기

이 프로젝트는 **GitHub Pages**로 배포하면, 로컬이 아닌 **웹 주소**로도 접속할 수 있습니다.

### 1. GitHub에 저장소 만들기

1. [GitHub](https://github.com) 로그인 후 **New repository** 클릭
2. 저장소 이름: `play_sim` (다른 이름을 쓰면 아래 2번 참고)
3. **Create repository** 클릭 (README 추가 안 해도 됨)

### 2. 로컬에서 Git 초기화 후 푸시

프로젝트 폴더에서:

```bash
git init
git add .
git commit -m "Initial commit: 물류 시뮬레이션 프로젝트"
git branch -M main
git remote add origin https://github.com/<본인_사용자명>/play_sim.git
git push -u origin main
```

`<본인_사용자명>`은 본인 GitHub 아이디로 바꾸세요.

### 3. GitHub Pages 켜기

1. GitHub 저장소 페이지에서 **Settings** → 왼쪽 메뉴 **Pages**
2. **Build and deployment**에서 **Source**를 **GitHub Actions**로 선택

이후 `main` 브랜치에 푸시할 때마다 자동으로 빌드·배포됩니다.  
(처음에 배포가 안 되면 **Actions** 탭에서 `Deploy to GitHub Pages` 워크플로우를 한 번 수동 실행해 보세요.)

### 4. 웹에서 접속 주소

배포가 끝나면 다음 주소로 접속할 수 있습니다.

- **주소**: `https://<본인_사용자명>.github.io/play_sim/`

예: 사용자명이 `hong`이면 `https://hong.github.io/play_sim/`

- 저장소 이름을 `play_sim`이 아닌 다른 이름(예: `logistics-sim`)으로 만들었다면:
  - `vite.config.ts`에서 `base: '/play_sim/'`를 `base: '/logistics-sim/'`처럼 저장소 이름에 맞게 수정한 뒤 푸시하세요.
