# 시뮬레이션 파트 — 필요한 정보 정리

시뮬레이션 구현을 위해 **이미 있는 정보**와 **추가로 정하면 좋은 정보**를 정리했습니다.

---

## 1. 이미 있는 정보 (추가 제공 불필요)

| 항목 | 출처 | 비고 |
|------|------|------|
| **배치·경로** | LayoutMap, pathUtils | 입고/출고/컨베이어/기계 위치, 경로 계산, 출력 포트 넘버링 |
| **창고** | layout.warehouseInventory | 품목별 초기 재고 |
| **레시피** | recipes.ts | 기계별 입력/출력 품목·수량, getDefaultRecipeIdForInput |
| **Tick·컨베이어** | 명세 §6, types.ts | TICK_SEC 0.5, CONVEYOR_TICKS_PER_MOVE 4, 처리 순서(출고→이동→기계입력→생산→기계출력→입고) |
| **기계 스펙** | equipmentSpecs.ts | 크기, 입출력 포트 개수·방향 |
| **전력** | powerConsumption.ts, 명세 §5 | 기계별 전력, 전력공급기 2×2·12×12 범위 |
| **출고 품목** | PlacedEquipment.outboundSelectedItem | 출고 포트별 창고에서 꺼낼 품목 |

이것만으로 **Phase A(출고·컨베이어·입고)** 와 **Phase B(기계 버퍼·레시피·전력)** 의 데이터·규칙은 구현 가능합니다.

---

## 2. 추가로 정하면 좋은 정보 (선택)

구현 시 **기본값을 두고 진행**할 수 있고, 나중에 바꿔도 됩니다.

| 항목 | 설명 | 적용값 |
|------|------|--------|
| **레시피 변환 시간** | 각 레시피의 `process_time_sec`. 미입력 시 `DEFAULT_PROCESS_TIME_SEC` 사용. | **2초** (`recipes.ts` 상수, 나중에 레시피별 수정 가능) |
| **출고 주기** | 출고 포트가 “몇 tick마다” 창고에서 1개 꺼내는지. | 매 tick 출고 시도(출력 셀 비어 있으면 1개 출고) |
| **다입력 레시피** | input_1 + input_2 필요 레시피. 입력 버퍼에 둘 다 충족될 때만 작업 시작할지. | 명세대로 “둘 다 충족 시에만” 작업 시작 |
| **시뮬 실행 속도** | 1x = 0.5초/tick 실시간, 2x/4x 등. | 1x(실시간) 고정 후, 나중에 배속 옵션 추가 |
| **시뮬 시간 상한** | 최대 진행 tick 또는 실시간. | 없음(무한) 또는 10분 등, 나중에 설정 UI로 추가 |

---

## 3. 정리

- **추가로 꼭 제공해야 할 정보는 없습니다.**  
  명세(§4·§5·§6)와 현재 데이터(배치·레시피·창고·경로·전력)만으로 시뮬레이션 로직 구현 가능합니다.
- **레시피 변환 시간**만 레시피 데이터에 값이 비어 있어,  
  초기에는 **공통 기본값(예: 5초)** 으로 두고 구현한 뒤,  
  원하시면 `recipes.ts`에 레시피별 `process_time_sec`를 채워 넣는 방식이 무난합니다.

원하시면 다음 단계로 **Phase A-1(Tick 엔진)** 부터 구체적인 구현 단계를 잡아 드리겠습니다.
